====  智能代理负载均衡系统 - 版本更新日志  =====

版本号: v1.1.0
发布日期: 2025-11-05 18:35
更新类型: 功能增强 (Feature Enhancement)

==  版本升级说明  ==

🎉 新增功能
-----------

1. 【核心功能】DNS映射管理系统
   - 新增DNS映射功能，支持在代理服务器层面进行自定义DNS解析
   - 无需修改系统hosts文件，即可实现域名到IP的自定义映射
   - 完美解决hosts配置导致的SOCKS5代理访问问题

2. 【数据库】新增dns_mappings表
   - 持久化存储DNS映射配置
   - 支持域名、IP地址、说明、启用状态等字段
   - 自动创建索引优化查询性能

3. 【后端API】DNS映射管理接口
   - GET    /api/dns-mappings           - 获取所有DNS映射
   - POST   /api/dns-mappings           - 创建DNS映射
   - PUT    /api/dns-mappings/:id       - 更新DNS映射
   - DELETE /api/dns-mappings/:id       - 删除DNS映射
   - PUT    /api/dns-mappings/:id/toggle - 切换启用状态

4. 【前端界面】DNS映射管理页面
   - 新增"DNS映射"菜单项
   - 表格形式展示所有DNS映射
   - 支持添加、编辑、删除、启用/禁用操作
   - 实时更新，修改后立即生效

5. 【代理服务器】DNS解析增强
   - 新增DNS映射缓存机制，提升解析性能
   - 自动加载和刷新DNS映射配置
   - 在SOCKS5握手时自动应用DNS重写
   - 控制台输出DNS重写日志，便于调试

✨ 功能优化
-----------

1. 性能优化
   - DNS映射采用内存缓存，避免频繁查询数据库
   - 修改配置后自动刷新缓存，无需重启服务

2. 用户体验优化
   - 统一使用SweetAlert2提示框，界面更美观
   - 操作成功后自动关闭提示（1.5秒）
   - 表单验证更完善，IP地址格式自动校验

3. 代码质量提升
   - 新增HTML转义函数，防止XSS攻击
   - 完善错误处理机制
   - 代码注释更详细

==  使用场景示例  ==

场景1: 测试环境访问
-------------------
问题: 需要访问测试环境的cms.zjzwfw.gov.cn，但不想修改系统hosts
解决:
  1. 在DNS映射中添加: cms.zjzwfw.gov.cn → 123.123.123.123
  2. 浏览器配置SOCKS5代理: 127.0.0.1:5678
  3. 直接访问 https://cms.zjzwfw.gov.cn
  4. 系统自动将域名解析到测试环境IP

场景2: 多环境快速切换
-------------------
问题: 需要在生产、测试、开发环境之间频繁切换
解决:
  1. 添加多个DNS映射（同一域名不同环境）
  2. 通过启用/禁用快速切换环境
  3. 无需修改hosts，无需重启浏览器

场景3: 团队协作
-------------------
问题: 团队成员需要统一的DNS配置
解决:
  1. 导出DNS映射配置（数据库备份）
  2. 团队成员导入配置
  3. 所有人使用相同的DNS解析规则

==  升级注意事项  ==

✅ 兼容性
---------
- 完全向后兼容，不影响现有功能
- 数据库自动升级，无需手动操作
- 现有代理配置不受影响

⚠️ 重要提示
-----------
1. 升级后首次启动会自动创建dns_mappings表
2. DNS映射功能默认不启用，需手动添加映射
3. 确保浏览器发送域名而非IP（不要配置hosts）
4. 查看控制台日志确认DNS重写是否生效

🔧 配置建议
-----------
1. 建议定期备份data/proxy.db数据库
2. 测试环境建议启用DNS映射日志
3. 生产环境建议限制DNS映射数量（性能考虑）
